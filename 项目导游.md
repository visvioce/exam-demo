## 📋 项目概览

这是一个**前后端分离**的在线考试系统，类似学校的网络考试平台。

| 部分 | 技术栈 | 作用 |
|------|--------|------|
| **前端** | Vue 3 + TypeScript + Element Plus | 用户看到的网页界面 |
| **后端** | Spring Boot + Java + MyBatis-Plus | 处理业务逻辑、存取数据 |
| **数据库** | MySQL | 存储所有数据（用户、题目、考试记录等） |

### 三种用户角色

系统里有三种人使用：
- **管理员(ADMIN)**：管用户、管轮播图、看所有数据
- **教师(TEACHER)**：出题、组卷、发布考试、批改主观题
- **学生(STUDENT)**：选课、参加考试、看成绩

## 🔐 第二站：用户认证模块（登录/注册）

> 这是几乎所有系统的第一个功能。理解它，你就理解了前后端如何"对话"。

### 核心流程

```
1. 用户输入用户名密码
2. 前端发送请求 → 后端验证 → 返回 JWT Token
3. 前端保存 Token，之后每次请求都带上它
4. 后端通过 Token 识别"你是谁"
```

### 前端相关文件

| 文件 | 作用 |
|------|------|
| [`frontend/src/views/auth/Login.vue`](frontend/src/views/auth/Login.vue) | 登录页面（左黑右白的漂亮界面） |
| [`frontend/src/views/auth/Register.vue`](frontend/src/views/auth/Register.vue) | 注册页面 |
| [`frontend/src/stores/auth.ts`](frontend/src/stores/auth.ts) | **登录状态管理**（Pinia Store） |
| [`frontend/src/api/auth.ts`](frontend/src/api/auth.ts) | 登录相关的 API 请求封装 |
| [`frontend/src/utils/request.ts`](frontend/src/utils/request.ts) | **HTTP请求工具**（重点看！） |

**重点看 `auth.ts`（Store）**：
- `token`：保存登录凭证（存在 localStorage 里，刷新页面不会丢失）
- `user`：保存当前用户信息
- `login()`：调用 API 登录，成功后保存 token
- `logout()`：清除登录状态

**重点看 `request.ts`**：
- 基于 Axios 封装
- **请求拦截器**：自动在请求头里加上 `Authorization: Bearer xxx`
- **响应拦截器**：统一处理错误，401 时自动跳转到登录页

### 后端相关文件

| 文件 | 作用 |
|------|------|
| [`backend/src/main/java/com/southcollege/exam/controller/AuthController.java`](backend/src/main/java/com/southcollege/exam/controller/AuthController.java) | 登录/注册接口 |
| [`backend/src/main/java/com/southcollege/exam/service/UserService.java`](backend/src/main/java/com/southcollege/exam/service/UserService.java) | 用户业务逻辑 |
| [`backend/src/main/java/com/southcollege/exam/utils/JwtUtil.java`](backend/src/main/java/com/southcollege/exam/utils/JwtUtil.java) | JWT Token 生成和验证 |
| [`backend/src/main/java/com/southcollege/exam/config/JwtInterceptor.java`](backend/src/main/java/com/southcollege/exam/config/JwtInterceptor.java) | 拦截请求，验证 Token |

**JWT 是什么？** 简单说就是一个"加密过的身份证"。用户登录后，后端发一个 Token 给前端，前端之后每次请求都带上这个 Token，后端就能知道"你是谁"而不用每次都查数据库。

## 📝 第五站：题库管理（教师功能）

### 题目类型

系统支持 **5 种题型**：

| 类型 | 英文名 | 说明 |
|------|--------|------|
| 单选题 | SINGLE_CHOICE | 4个选项，选1个 |
| 多选题 | MULTIPLE_CHOICE | 多个选项，选多个 |
| 判断题 | TRUE_FALSE | 正确/错误 |
| 填空题 | FILL_BLANK | 填写答案，可多空 |
| 简答题 | ESSAY | 文字作答，需教师批改 |

### 数据库表

**文件**：[`database/schema.sql`](database/schema.sql) → `questions` 表

```sql
CREATE TABLE questions (
    id BIGINT PRIMARY KEY,
    content TEXT,           -- 题目内容（支持HTML）
    type ENUM(...),         -- 题目类型
    difficulty ENUM(...),   -- 难度：EASY/MEDIUM/HARD
    score DECIMAL,          -- 分值
    teacher_id BIGINT,      -- 出题教师
    options JSON,           -- 选项（选择题）
    correct_answer JSON,    -- 正确答案
    scoring_criteria JSON,  -- 评分标准（简答题）
    explanation TEXT        -- 答案解析
);
```

### 后端实体

**文件**：[`backend/src/main/java/com/southcollege/exam/entity/Question.java`](backend/src/main/java/com/southcollege/exam/entity/Question.java)

注意 `options`、`correctAnswer`、`scoringCriteria` 这三个字段：
- 数据库里存的是 JSON 字符串
- Java 里用 `JacksonTypeHandler` 自动转成对象/列表

### AI 出题功能（亮点！）

**文件**：[`backend/src/main/java/com/southcollege/exam/service/AiQuestionService.java`](backend/src/main/java/com/southcollege/exam/service/AiQuestionService.java)

教师可以调用 AI 自动生成题目！流程：
1. 教师填写科目、题型、难度、数量
2. 后端构建 Prompt（提示词）
3. 调用 OpenAI 格式的 API（如 ChatGPT）
4. AI 返回 JSON 格式的题目
5. 后端解析并保存到题库

支持**流式生成**（SSE），可以实时看到 AI 在"打字"生成题目。

**前端调用**：[`frontend/src/api/ai.ts`](frontend/src/api/ai.ts)

## 📄 第六站：试卷管理

### 什么是试卷？

试卷 = 一个题目列表 + 每题的分值配置

组卷有两种方式：
- **手动组卷**：教师从题库一道道挑选题目
- **自动组卷**：设置规则（如"单选10道、每道2分"），系统自动从题库随机抽取

### 数据库表

```sql
CREATE TABLE papers (
    id BIGINT PRIMARY KEY,
    name VARCHAR(200),      -- 试卷名称
    course_id BIGINT,       -- 所属课程
    teacher_id BIGINT,      -- 创建教师
    questions JSON,         -- 题目配置 [{questionId, score}, ...]
    total_score DECIMAL,    -- 总分
    type ENUM('MANUAL', 'AUTO'),  -- 组卷方式
    status ENUM('DRAFT', 'PUBLISHED', 'ARCHIVED')
);
```

## ⏰ 第七站：考试管理（核心功能）

### 考试生命周期

```
草稿(DRAFT) → 发布(PUBLISHED) → 进行中(STARTED) → 结束(ENDED)
                    ↓
                取消(CANCELLED)
```

### 核心流程

1. **教师创建考试**：设置标题、时间、选择试卷
2. **教师发布考试**：学生才能看到
3. **学生参加考试**：进入 [`ExamTake.vue`](frontend/src/views/exam/ExamTake.vue)
4. **学生提交**：客观题自动评分
5. **教师批改**：主观题（简答题）需要教师手动给分
6. **查看成绩**

### 考试列表页面

**文件**：[`frontend/src/views/exam/ExamList.vue`](frontend/src/views/exam/ExamList.vue)

这是考试的"入口页面"，不同角色看到不同内容：

**学生视角**：
- 显示"我的考试"列表
- 显示考试状态：未开始/进行中/已结束/已取消
- 操作按钮：参加考试（仅在时间内可点）、查看回顾

**教师/管理员视角**：
- 显示所有考试列表
- 支持按课程、状态筛选
- 操作按钮：查看、编辑（仅草稿）、删除（仅草稿）、发布、取消
- 分页显示

### 考试详情页面

**文件**：[`frontend/src/views/exam/ExamDetail.vue`](frontend/src/views/exam/ExamDetail.vue)

点击考试标题进入的详情页：
- 显示考试基本信息（名称、时间、时长、总分）
- 显示试卷信息（名称、类型、题目数量）
- 显示题目列表（序号、内容、题型、分值）
- 教师可查看考试记录按钮

### 考试页面（重点看！）

**文件**：[`frontend/src/views/exam/ExamTake.vue`](frontend/src/views/exam/ExamTake.vue)

这是一个**全屏独立页面**（没有侧边栏），包含：
- 顶部：考试信息 + 倒计时
- 左侧：题目列表（单选/多选/判断/填空/简答）
- 右侧：答题卡（显示哪些题已答）
- 自动保存：每30秒自动保存答案到服务器
- 本地备份：同时存到 localStorage，防止浏览器崩溃

### 后端考试服务（重点看！）

**文件**：[`backend/src/main/java/com/southcollege/exam/service/ExamService.java`](backend/src/main/java/com/southcollege/exam/service/ExamService.java)

这是整个系统**最复杂的文件**（1300+行），核心方法：

| 方法 | 作用 |
|------|------|
| `startExam()` | 学生开始考试，创建考试记录 |
| `autoSaveExam()` | 自动保存答案 |
| `submitExam()` | 提交考试，客观题自动评分 |
| `gradeSubjectiveAnswers()` | 教师给主观题评分 |
| `getExamResult()` | 查询考试结果 |

**自动评分逻辑**：
- 单选题：字符串比较（忽略大小写）
- 多选题：排序后比较数组
- 判断题：兼容 "A/正确/true" 和 "B/错误/false"
- 填空题：支持多个正确答案、忽略空白、支持数字等价比较
- 简答题：不自动评分，标记为待批改

## 🔧 第九站：后端架构详解

后端采用经典的三层架构：

### 1. Controller 层（接收请求）

**示例**：[`AuthController.java`](backend/src/main/java/com/southcollege/exam/controller/AuthController.java)

```java
@RestController
@RequestMapping("/api/auth")
public class AuthController {
    @PostMapping("/login")
    public Result<LoginResponse> login(@RequestBody LoginRequest request) {
        // 调用 Service，返回统一格式
    }
}
```

### 2. Service 层（业务逻辑）

处理具体业务，如考试评分、权限检查等。

### 3. Mapper 层（数据库访问）

MyBatis-Plus 自动生成，基本不用写 SQL。

### 4. Entity（数据模型）

**示例**：[`User.java`](backend/src/main/java/com/southcollege/exam/entity/User.java)

```java
@Data
@TableName("users")
public class User {
    @TableId(type = IdType.AUTO)
    private Long id;
    private String username;
    private String password;  // BCrypt加密
    private String role;      // ADMIN/TEACHER/STUDENT
}
```

### 5. DTO（数据传输对象）

- `dto/request/`：前端传给后端的数据
- `dto/response/`：后端返回给前端的数据

### 6. 安全配置

**文件**：[`SecurityConfig.java`](backend/src/main/java/com/southcollege/exam/config/SecurityConfig.java)

- 禁用 CSRF（因为用 JWT）
- 配置 CORS（允许前端跨域访问）
- 白名单：登录接口、Swagger 文档

## 🎓 第十站：课程管理

> 考试是基于课程的，学生必须先加入课程才能参加考试。

### 核心概念

- **课程（Course）**：教师创建的班级/课程
- **课程成员（CourseMember）**：学生加入课程的记录
- 一个学生可以加入多个课程，一个课程可以有多个学生

### 前端相关文件

| 文件 | 作用 |
|------|------|
| [`frontend/src/views/course/CourseList.vue`](frontend/src/views/course/CourseList.vue) | 课程列表（教师管理/学生选课） |
| [`frontend/src/views/course/CourseDetail.vue`](frontend/src/views/course/CourseDetail.vue) | 课程详情（成员列表、考试列表） |
| [`frontend/src/api/course.ts`](frontend/src/api/course.ts) | 课程 API 封装 |

### 功能说明

**教师视角**：
- 创建课程（填写名称、代码、学分、描述）
- 管理课程成员（查看已加入的学生）
- 查看课程下的考试

**学生视角**：
- 浏览所有课程
- 加入/退出课程
- 查看已加入的课程和考试

### 数据库表

```sql
CREATE TABLE courses (
    id BIGINT PRIMARY KEY,
    name VARCHAR(200),      -- 课程名称
    code VARCHAR(50),       -- 课程代码
    description TEXT,       -- 课程描述
    credits INT,            -- 学分
    teacher_id BIGINT,      -- 授课教师
    status ENUM('ACTIVE', 'INACTIVE')
);

CREATE TABLE course_members (
    id BIGINT PRIMARY KEY,
    course_id BIGINT,       -- 课程ID
    student_id BIGINT,      -- 学生ID
    status ENUM('ACTIVE', 'DROPPED')
);
```

## 💡 小白常见问题

### Q1: Vue 文件里的 `<template>`、`<script>`、`<style>` 是什么？
- `<template>`：HTML 结构（页面长什么样）
- `<script setup lang="ts">`：TypeScript 逻辑（页面怎么动）
- `<style scoped>`：CSS 样式（页面什么颜色、多大）

### Q2: `ref`、`computed`、`onMounted` 是什么？
- `ref`：定义一个"响应式变量"（值变了页面自动更新）
- `computed`：计算属性（根据其他值自动计算）
- `onMounted`：页面加载完成后执行的代码

### Q3: 前端怎么调用后端接口？
通过 Axios 发送 HTTP 请求：
```typescript
// 前端
const res = await request.post('/auth/login', { username, password })
// res.data 就是后端返回的数据
```

### Q4: 后端怎么接收前端请求？
```java
// 后端
@PostMapping("/login")
public Result<LoginResponse> login(@RequestBody LoginRequest request) {
    // request 就是前端传来的数据
}
```

### Q5: `@TableName`、`@TableId` 是什么？
这是 MyBatis-Plus 的注解：
- `@TableName("users")`：这个类对应数据库的 `users` 表
- `@TableId`：这是主键字段
- `@TableLogic`：逻辑删除标记（0=未删除，1=已删除）
